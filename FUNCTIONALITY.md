# Трекер задач — Функционал для пользователя

## Авторизация
- Вход по логину и паролю
- Два роли: **Руководитель** (admin) и **Сотрудник**
- Сессия сохраняется — при повторном входе пароль вводить не нужно
- Выход из системы

---

## Доска задач (Kanban)

Три колонки:

| Колонка | Описание |
|---------|----------|
| **Срочные** | Задачи с высоким приоритетом и просроченные задачи |
| **В работе** | Задачи в процессе выполнения |
| **Выполнено** | Завершённые задачи |

### Отображение задачи
- **Цвет левой полоски** — цвет исполнителя
- **Стрелка ↗** — задача делегирована руководителем
- **Стрелка ↙** — задача делегирована сотрудником
- **Приоритет** — высокий (красный), средний (жёлтый), низкий (зелёный)
- Сортировка в колонке: сначала высокий приоритет, потом средний, потом низкий

### Действия над задачей
- **✅ Выполнить** — перенести в «Выполнено»
- **↩ Вернуть** — вернуть из «Выполнено» в предыдущий статус
- **📤 Делегировать** — назначить другому сотруднику
- **⭕ Открыть** — подробная информация о задаче
- **⚙️ Настройки** — редактирование (только руководитель или создатель)
- **🗑 Удалить** — удалить задачу (только руководитель)

### Drag & Drop
Задачи можно перетаскивать между колонками мышью.

## Создание / редактирование задачи

| Поле | Описание |
|------|----------|
| Заголовок * | Обязательное поле |
| Описание | Текст описания задачи |
| Статус | Срочные / В работе / Выполнено |
| Приоритет | Низкий / Средний / Высокий |
| Срок выполнения | Дата и время (шаг 15 минут) |
| Исполнитель | Назначение из списка сотрудников |

---

## Делегирование

- Руководитель может делегировать задачу **любому сотруднику** (включая себя)
- Сотрудник может делегировать задачу **другому сотруднику** (но не руководителю)
- При делегировании исполнитель получает уведомление

### Обозначения
- **↗** (синяя стрелка) — руководитель делегировал сотруднику
- **↙** (красная стрелка) — сотрудник делегировал другому сотруднику

---

## Уведомления

### В приложении (toast-уведомления)
- **Новая задача** — при назначении задачи (звук + всплывающее окно)
- **Делегирование** — при делегировании задачи (звук + всплывающее окно)
- **Просрочка** — в день истечения срока (звук + всплывающее окно)
- Уведомления **не исчезают автоматически** — закрываются крестиком вручную

### По email (EmailJS)
- При **создании задачи** с назначенным исполнителем — письмо исполнителю
- При **делегировании** — письмо новому исполнителю
- При **истечении срока** (последние 24 часа):
  - Письмо исполнителю
  - Письмо руководителю

> Для email-уведомлений у пользователя должен быть заполнен email в профиле.

## Автопереход просроченных задач

- Проверка каждую **1 минуту**
- Если время истекло и задача в статусе «В работе» → автоматически переводится в «Срочно»
- Приоритет задачи **сохраняется**

---

## Управление сотрудниками (руководитель)

- Добавление сотрудника (логин, пароль, email, цвет)
- Редактирование сотрудника (логин, роль, пароль, email, цвет)
- Изменение роли: Руководитель / Сотрудник
- Смена логина обновляет ссылки во всех задачах

---

## Экспорт / импорт Excel

- **Экспорт** — выгрузка всех задач в .xlsx файл
- **Импорт** — загрузка задач из .xlsx файла
- Поддерживаемые поля: ID, Заголовок, Описание, Статус, Создал, Исполнитель, Приоритет, Срок, Делегировано, Создано, Обновлено

---

## Технические особенности

- **PWA** — установка как приложение на рабочий стол
- **Firebase Realtime Database** — синхронизация между устройствами в реальном времени
- **Firebase Auth** — аутентификация
- **Работает офлайн** — данные кэшируются

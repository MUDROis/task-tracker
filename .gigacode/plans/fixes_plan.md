# План исправлений для трекера задач

## Описание проблем и план исправлений

### 1. Кнопка "Выйти" съехала вниз
**Проблема**: Кнопка "Выйти" В МОБИЛЬНОЙ ВЕРСИИ расположена не напротив слова "Руководитель", а смещена вниз.

**Файл**: `style.css`
**Исправление**:
- В блоке `.app-header` убедиться, что `.header-left` и `.header-right` выровнены по центру
- Проверить свойства `align-items: center` для `.app-header`
- Убедиться, что `.header-right` не имеет лишних отступов сверху/снизу
- Удалить `margin-left: auto` из `.header-right`, если оно вызывает неправильное позиционирование

### 2. Адаптивность ширины приложения
**Проблема**: Ширина приложения не адаптируется к ширине экрана.

**Файл**: `style.css`
**Исправление**:
- В блоке `#app` заменить `max-width: 1400px` на `width: 100%` или `max-width: 100%`
- Убедиться, что нет фиксированных значений ширины в других блоках
- Добавить `box-sizing: border-box` ко всем элементам (уже есть в reset)

### 3. Задачи в раздел "Срочные" не попадают
**Проблема**: Задачи с высоким приоритетом не попадают в раздел "Срочные", даже если приоритет высокий.

**Файл**: `app.js`
**Исправление**:
- В функции `renderBoard()` добавить логику: если задача имеет `priority === 'high'`, то автоматически ставить `status = 'urgent'`
- Либо при ручном перетаскивании задачи в раздел "Срочные", проверять, что задача действительно попадает туда
- Проверить функцию `changeStatus()` - возможно, она не позволяет менять статус на 'urgent'

### 4. Всплывающее окно выбора даты не адаптируется
**Проблема**: При назначении срока в задаче всплывающее окно (где выбираются день, месяц и год) не адаптируется к ширине экрана - по вертикали год не видно, кнопки "Установить" тоже не видны.

**Файл**: `style.css`
**Исправление**:
- В блоке `.modal-content` добавить `max-width: 90vw` вместо фиксированного `max-width: 500px`
- Добавить `overflow: auto` для прокрутки в мобильной версии
- В медиа-запросе `@media (max-width: 768px)` уменьшить `padding` внутри `.modal-content` (с `2rem` до `1rem` или `0.75rem`)

### 5. Цвет исполнителя не отображается
**Проблема**: В списке исполнителей/ролей цветовой кружок отображается синим для всех ролей, хотя должен отображать выбранный цвет.

**Файл**: `app.js`
**Исправление**:
- В функции `renderUsersList()` проверить, что цвет берется из `u.color`, а не жестко задается
- В блоке `.user-color-dot` убедиться, что `style="background:' + (u.color || '#94a3b8') + '"` правильно работает
- Проверить функцию `createTaskCard()` - цвет исполнителя должен браться из `assigneeUser.color`
- Убедиться, что цвет сохраняется при создании пользователя (в `addUserForm` и `editUserForm`)

### 6. Высота пустых блоков слишком большая
**Проблема**: Пустые блоки (Срочные, В работе, Выполнено) слишком высокие, занимают много места.

**Файл**: `style.css`
**Исправление**:
- В блоке `.column` уменьшить `min-height: 300px` до `min-height: 80px`
- В блоке `.task-list` уменьшить `min-height: 100px` до `min-height: 50px`
- При пустом списке задач отображать только узкую полоску с названием

### 7. Красная заливка в блоке "Срочные"
**Проблема**: В блоке "Срочные" есть красная заливка `#fef2f2`, которая раздражает.

**Файл**: `style.css`
**Исправление**:
- В блоке `.column[data-status="urgent"]` изменить `background: #fef2f2` на `background: #f8fafc` (такой же как у других блоков)
- Убрать `border: 1px solid #fecaca`
- Убрать изменение `border-bottom-color` в `.column[data-status="urgent"] .column-header`

---

## Приоритет исправлений

1. **Критичный**: №3 (Задачи не попадают в раздел "Срочные") - нарушает основную функциональность
2. **Высокий**: №5 (Цвет исполнителя), №7 (Красная заливка) - влияют на UX
3. **Средний**: №1 (Кнопка "Выйти"), №4 (Всплывающее окно даты) - влияют на адаптивность
4. **Низкий**: №2 (Адаптивность ширины), №6 (Высота пустых блоков) - косметические правки

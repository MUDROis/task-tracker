# Трекер задач

PWA-приложение для управления задачами команды с доской Kanban, делегированием и email-уведомлениями.

## Быстрый старт

1. Откройте `index.html` в браузере
2. Войдите под учётной записью **admin** / **admin**
3. Готово — доска с тремя колонками перед вами

## Роли

| Роль | Возможности |
|------|-------------|
| **Руководитель** (admin) | Всё: создание, редактирование, делегирование, управление сотрудниками, удаление любых задач |
| **Сотрудник** | Создание задач, выполнение/возврат своих задач, редактирование своих задач, drag & drop |

## Доска Kanban

Три колонки:

- **Срочные** — задачи с высоким приоритетом, требующие внимания
- **В работе** — задачи в процессе выполнения
- **Выполнено** — завершённые задачи

Задачи можно перетаскивать между колонками (drag & drop).

## Задачи

### Создание
Нажмите **➕ Новая задача** и заполните:
- **Заголовок** — название задачи (обязательно)
- **Описание** — детали (необязательно)
- **Приоритет** — низкий / средний / высокий
- **Срок выполнения** — дата дедлайна
- **Исполнитель** — назначить сотрудника

### Редактирование
Дважды кликните по карточке задачи.

### Действия на карточке
- **✅ Выполнить** — переместить в "Выполнено"
- **↩ Вернуть** — вернуть из "Выполнено" в исходную колонку
- **📤 Делегировать** — назначить сотрудника (только для руководителя). На карточке появится стрелка **↗** в правом верхнем углу
- **🗑 Удалить** — удалить задачу (с подтверждением)

### Цвет исполнителя
Левая полоска карточки окрашивается в цвет назначенного сотрудника.

## Делегирование

Нажмите **📤 Делегировать** на любой задаче (кроме выполненных). Выберите сотрудника из списка. Сотрудник получит email-уведомление о делегированной задаче.

Стрелка **↗** на карточке означает, что задача была делегирована через эту кнопку.

## Управление сотрудниками

Доступно только руководителю. Нажмите **👥 Управление** в панели инструментов.

### Добавление сотрудника
Заполните логин, пароль, email для уведомлений и выберите цвет на доске.

### Редактирование сотрудника
Нажмите **Изменить** у сотрудника. Можно изменить:
- Логин (обновится во всех задачах)
- Пароль (оставьте пустым без изменений)
- Email для уведомлений
- Цвет на доске (color-picker + палитра)

### Удаление сотрудника
Нажмите **Удалить**. Если удаляете себя — произойдёт выход из системы.

## Уведомления по почте

Используется [EmailJS](https://www.emailjs.com) (бесплатно до 200 писем/мес).

### Настройка
1. Зарегистрируйтесь на [emailjs.com](https://www.emailjs.com)
2. Создайте **Email Service** (Gmail, Outlook и т.д.)
3. Создайте **Email Template** с переменными:
   - `{{to_email}}` — адрес получателя
   - `{{to_name}}` — имя получателя
   - `{{task_title}}` — название задачи
   - `{{from_name}}` — кто делегировал
4. Впишите ключи в `app.js`:
   ```js
   const EMAILJS_PUBLIC_KEY = 'ваш ключ';
   const EMAILJS_SERVICE_ID = 'ваш service id';
   const EMAILJS_TEMPLATE_ID = 'ваш template id';
   ```
5. Укажите email у каждого сотрудника в профиле

Уведомление отправляется при делегировании задачи. Если email не указан — уведомление не отправляется.

## Экспорт и импорт Excel

### Экспорт
Нажмите **📤 Выгрузить Excel** — скачается файл `.xlsx` со списком всех задач.

### Импорт
Нажмите **📥 Импорт Excel** и выберите файл `.xlsx`. Задачи с совпадающим ID будут обновлены, новые — добавлены.

### Колонки файла
| Колонка | Описание |
|---------|----------|
| ID | Уникальный идентификатор |
| Заголовок | Название задачи |
| Описание | Текст задачи |
| Статус | Срочно / В работе / Выполнено |
| Создал | Кто создал |
| Исполнитель | Кому назначена |
| Приоритет | low / medium / high |
| Срок | Дата дедлайна |
| Создано | Дата создания |
| Обновлено | Дата последнего изменения |

## Технологии

- **HTML/CSS/JS** — без фреймворков
- **PWA** — Service Worker для работы без интернета
- **localStorage** — хранение данных в браузере
- **SheetJS** — экспорт/импорт Excel
- **EmailJS** — email-уведомления

## Структура файлов

```
task-tracker/
├── index.html          — разметка приложения
├── app.js              — вся логика
├── style.css           — стили
├── manifest.json       — PWA-манифест
├── service-worker.js   — кеширование для offline
└── README.md           — эта инструкция
```

## Примечания

- Данные хранятся в `localStorage` браузера. При очистке браузера данные удаляются
- Рекомендуется регулярно экспортировать задачи в Excel как бэкап
- При первом открытии приложение кеширует все скрипты и работает без интернета
